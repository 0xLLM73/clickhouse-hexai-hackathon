# Security Policy

This is a fast-moving public hackathon repository.

## During the Sprint

- Do not commit secrets, credentials, private datasets, or customer data.
- Use `.env` for local secrets and keep `.env.example` limited to placeholder names.
- Prefer short-lived or least-privilege credentials for demos.
- If a secret is exposed, rotate it immediately before continuing work.

## Reporting

If you find a security issue during the hackathon, contact the project owner directly instead of opening a public issue with sensitive details.
