# Contributing

## Hackathon Mode

Speed matters, but avoid changes that make the project unsafe to publish.

## Basic Rules

- Keep commits focused and descriptive.
- Do not commit local secrets, credentials, or private data.
- Update `.env.example` when adding a required environment variable.
- Add setup notes to `README.md` as soon as they become necessary.
- Use pull requests for high-risk changes, but do not block small sprint work on process.

## Before Pushing

- Run the relevant local check if one exists.
- Scan your diff for secrets and generated data.
- Make sure public demo data is synthetic or approved for release.
